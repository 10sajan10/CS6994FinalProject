Metrics and results guide
=========================

Metric meaning
--------------

RMSE:

- Root mean squared error in the target variable's native unit.
- Lower is better.

MAE:

- Mean absolute error in the target variable's native unit.
- Lower is better.

Persistence baseline:

- Uses the latest observed value in the lookback window as the forecast for all
  24 future horizons.
- discharge baseline value: discharge_t-1.
- air-temperature baseline value: air_temp_t-1.

Skill vs persistence:

- skill = 1 - model_error / persistence_error
- Positive skill means the model beats persistence.
- Zero means tied with persistence.
- Negative skill means worse than persistence.

Normal vs season-specific interpretation
----------------------------------------

Discharge:

- Season-specific training improves many individual site/season/horizon groups.
- It is strongest for winter, fall, and summer discharge.
- Spring discharge is weaker, especially for site 4, and can pull the pooled
  average down.
- Best use: keep season-specific discharge models as candidates for inference,
  especially outside spring, but compare against the normal all-season model
  before deployment.

Air temperature:

- For site 1, the normal all-season air-temperature model is better overall than
  the season-specific models.
- Site 2 has season-specific metrics, but the committed normal air-temperature
  metrics currently only include site 1, so site 2 does not yet have a direct
  normal-vs-season-specific comparison.

Comparison note:

- Normal metric files group season by target timestamp.
- Season-specific metric files group season by the trained forecast-start season.
- For a strict normal-vs-season-specific comparison, derive season from
  target_timestamp in the season-specific prediction CSV.
