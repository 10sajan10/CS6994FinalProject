Streamflow Parquet V2 Feature Guide

Purpose
This file explains the parquet datasets generated by
build_streamflow_training_parquet_v2_multihorizon.ipynb.

Expected parquet outputs
- ./CS6994FinalProject/streamflow_parquet_v2/discharge_training_site_3_lb168_air24avg_h24.parquet
-./CS6994FinalProject/streamflow_parquet_v2/discharge_training_site_4_lb168_air24avg_h24.parquet

What each parquet file is
Each parquet file contains supervised training rows for one target stream site.
Each row uses:
- 168 hours of past discharge history
- 24-hour average air temperature context
- latest snow depth context
- calendar features
- 24 future hourly discharge targets

Target sites
- Site 3: Logan River at the Utah Water Research Laboratory west bridge
- Site 4: Logan River at Mendon Road (600 South)

Source sites used for features
- Discharge history: the target stream site itself
- Air temperature: climate site 2
- Snow depth: climate site 1

Main metadata columns
- site_id
  Target stream site identifier for the row.
- history_end_time
  Timestamp of the last hour in the 168-hour lookback window.
- forecast_start_time
  First hour being predicted.
- forecast_end_time
  Last hour in the 24-hour prediction horizon.

Context feature columns
- snow_depth_latest
  Most recent snow depth value available at the end of the history window.
- air_temp_avg_last_24h
  Average air temperature over the 24 hours before forecast start.
- year
  Forecast start year.
- month
  Forecast start month number.
- day
  Forecast start day of month.
- hour
  Forecast start hour.
- day_of_week
  Forecast start weekday, where Monday is 0.
- day_of_year
  Forecast start day number within the year.
- season
  Encoded season:
  0 = winter
  1 = spring
  2 = summer
  3 = fall
- sample_stride_hours
  Sampling stride used when that row was created.
  1 hour for March through June.
  6 hours for all other months.

History feature columns
- discharge_t-168 through discharge_t-1
  Hourly discharge history from oldest to most recent.
  Example:
  discharge_t-168 = discharge value 168 hours before forecast start
  discharge_t-1 = discharge value 1 hour before forecast start

Target columns
- target_discharge_t+1 through target_discharge_t+24
  Observed future discharge values for the next 24 hours.
  Example:
  target_discharge_t+1 = discharge 1 hour after forecast start
  target_discharge_t+24 = discharge 24 hours after forecast start

Data quality rules used to build rows
- Negative discharge values are treated as missing.
- Negative snow depth values are treated as missing.
- Negative air temperature values are allowed.
- Missing discharge and air temperature inputs are filled with time interpolation plus edge fill.
- Missing snow depth is filled using carry-forward and backfill.
- Target discharge values must be observed, not imputed.
- Any sample is dropped if discharge, air temperature, or snow depth has a consecutive raw missing gap of 10 hours or more within the full history-plus-forecast window.

Row generation rules
- Default stride is 6 hours.
- For forecast start months March, April, May, and June, stride is 1 hour to generate denser spring runoff examples.

How to use these files
- Use one parquet per site for site-specific modeling.
- Combine both parquet files if you want a multi-site training dataset.
- Treat discharge_t-* columns as encoder/history inputs.
- Treat target_discharge_t+* columns as forecast labels.
